= Secure Coding Agents on OpenShift
include::_attributes.adoc[]

Welcome to the **Secure Coding Agents on OpenShift** workshop.

In this hands-on workshop, you will learn why AI coding agents need sandboxed execution, how the industry approaches this problem, and how to build a production-grade solution using **Red Hat OpenShift Dev Spaces** and **Kubernetes Agent Sandbox** with **Kata VM isolation**.

== What You Will Learn

By the end of this workshop, you will:

* Understand the security risks of AI agents executing code with developer-level permissions
* Know how Anthropic, Google, GitHub, and OpenAI each solve the sandboxing problem
* Deploy a secure Agent Sandbox environment on OpenShift using a Goose AI recipe
* Use Goose (an open-source AI coding agent) inside a Dev Spaces workspace
* Prove that sandbox isolation blocks credential theft, data exfiltration, and API access
* Generate, deploy, and test an MCP server on OpenShift — end to end

== Workshop Flow

This workshop follows a *Why -> What -> How -> Do* progression:

[cols="1,3"]
|===

|===

== Prerequisites

You need:

* Access to an **OpenShift 4.14+** cluster with `cluster-admin` permissions
* **Red Hat OpenShift Dev Spaces** installed on the cluster
* The `oc` CLI installed on your local machine (or use the terminal inside Dev Spaces)
* A web browser

=== Verify Cluster Access

[.console-input]
[source,bash]
----
oc whoami
----

[.console-output]
[source,bash]
----
admin
----

[.console-input]
[source,bash]
----
oc version
----

[.console-output]
[source,bash]
----
Client Version: 4.14.x
Server Version: 4.14.x
Kubernetes Version: v1.28.x
----

You should see your OpenShift server version (4.14 or later).

=== Verify Dev Spaces

[.console-input]
[source,bash]
----
oc get pods -n openshift-devspaces | head -5
----

[.console-output]
[source,bash]
----
NAME                                  READY   STATUS    RESTARTS   AGE
devspaces-operator-xxxxx-yyyyy        1/1     Running   0          2d
postgres-xxxxx-yyyyy                  1/1     Running   0          2d
----

You should see Dev Spaces operator pods running.
