Setting Up the Environment

Overview

We’ll deploy the full Agent Sandbox environment using a Goose AI recipe. This demonstrates production-grade infrastructure-as-code via AI agent.

Option A: Using the Goose Recipe (Recommended)

Run the deploy-agent-sandbox recipe:

goose run --recipe skills/deploy-agent-sandbox.yaml
The recipe defaults to namespace sandbox-devspaces and 3 warm pool replicas. Override with --params: goose run --recipe skills/deploy-agent-sandbox.yaml --params namespace=my-ns --params warmpool_replicas=5

The recipe performs the following steps:

  • Installs the agent-sandbox operator (if not already present)

  • Creates the target namespace and grants OpenShift SCC permissions

  • Deploys a SandboxTemplate with Kata VM isolation

  • Applies a NetworkPolicy for deny-all egress (DNS only)

  • Creates a WarmPool with 3 replicas for instant sandbox allocation

  • Verifies CRDs, controller, RuntimeClass, and warm pool pods

Option B: Manual Setup

For those who prefer scripts, run:

./demo/setup.sh

Or perform the steps manually:

  1. Install agent-sandbox operator and extensions:

    oc apply -f https://github.com/kubernetes-sigs/agent-sandbox/releases/download/{agent-sandbox-version}/manifest.yaml
oc apply -f https://github.com/kubernetes-sigs/agent-sandbox/releases/download/{agent-sandbox-version}/extensions.yaml

  2. Create the namespace:

    oc create namespace {sandbox-namespace}

  3. (OpenShift only) Grant SCC permissions so the controller can create pods with the specified runAsUser:

    oc adm policy add-scc-to-user anyuid -z agent-sandbox-controller -n agent-sandbox-system
oc adm policy add-scc-to-user anyuid -z default -n {sandbox-namespace}
    On vanilla Kubernetes this step is not needed. OpenShift’s restricted-v2 SCC rejects pods whose runAsUser falls outside the namespace’s auto-assigned UID range.

  4. Apply the SandboxTemplate:

    oc apply -f demo/sandbox-template.yaml

  5. Apply the WarmPool:

    oc apply -f demo/sandbox-warmpool.yaml

  6. Apply the NetworkPolicy for egress isolation:

    oc apply -f demo/sandbox-networkpolicy.yaml
    The SandboxTemplate defines a networkPolicy section, but the agent-sandbox controller (v0.2.1) does not auto-create Kubernetes NetworkPolicy resources from it. This manual step ensures the deny-all egress (DNS only) policy is enforced.

Understanding the SandboxTemplate

The SandboxTemplate defines the security posture for each sandbox pod:

# Source of truth: demo/sandbox-template.yaml -- keep in sync
apiVersion: extensions.agents.x-k8s.io/v1alpha1
kind: SandboxTemplate
metadata:
  name: secure-agent-sandbox
  namespace: sandbox-devspaces
spec:
  podTemplate:
    spec:
      # ── Runtime isolation ──────────────────────────────────────────────
      # Choose ONE based on your environment:
      #   kata-remote  → OpenShift on AWS (peer-pods, each sandbox = dedicated EC2 VM)
      #   kata         → OpenShift on bare metal (each sandbox = local lightweight VM)
      #   gvisor       → GKE (kernel-level syscall filtering)
      # Remove this field entirely to use the default container runtime (runc).
      runtimeClassName: kata-remote

      automountServiceAccountToken: false
      securityContext:
        runAsUser: 10001
        runAsGroup: 0
        runAsNonRoot: true
      containers:
        - name: agent-sandbox
          image: quay.io/rbrhssa/mcp-skills:latest
          command: ["/bin/sh", "-c", "sleep infinity"]
          resources:
            limits:
              cpu: "2"
              memory: 4Gi
            requests:
              cpu: 500m
              memory: 1Gi
          ports:
            - containerPort: 8080
              protocol: TCP
          env:
            - name: HOME
              value: /home/user
    metadata:
      annotations:
        # ── Kata hypervisor tuning (only used when runtimeClassName is kata or kata-remote) ──
        io.katacontainers.config.hypervisor.default_vcpus: "2"
        io.katacontainers.config.hypervisor.default_memory: "4096"
        # ── AWS peer-pods ONLY (remove on bare metal / GKE) ──
        # Forces the EC2 instance type. Required because the peer-pods AMI uses TPM 2.0,
        # which older instance families (t2) do not support.
        io.katacontainers.config.hypervisor.machine_type: t3.medium
        io.kubernetes.cri-o.userns-mode: ""

  # NetworkPolicy: deny all except DNS and ingress from DevSpaces workspace pods.
  # NOTE: As of agent-sandbox v0.2.1, this section is stored in the template but
  # NOT auto-applied as a Kubernetes NetworkPolicy. You must also apply
  # demo/sandbox-networkpolicy.yaml separately. See Module 4 in the workshop.
  networkPolicy:
    ingress:
      - from:
          - namespaceSelector:
              matchLabels:
                kubernetes.io/metadata.name: sandbox-devspaces
    egress:
      - ports:
          - protocol: UDP
            port: 53
          - protocol: TCP
            port: 53

Core Settings (all environments)

Setting Purpose

automountServiceAccountToken: false

Prevents Kubernetes API access — the sandbox cannot read secrets or call the cluster API.

runAsNonRoot: true / runAsUser: 10001

Runs as non-root user. On OpenShift, granting anyuid SCC is required because the hardcoded UID 10001 falls outside the namespace’s auto-assigned range.

resources.limits

CPU and memory caps per sandbox (2 CPU, 4Gi memory in this template).

networkPolicy

Deny-all egress except DNS (ports 53). Ingress only from the workspace namespace. Prevents data exfiltration and lateral movement.
As of agent-sandbox v0.2.1, the controller does not auto-create Kubernetes NetworkPolicy resources from the template’s networkPolicy section. You must apply demo/sandbox-networkpolicy.yaml separately (step 6 above). This may be fixed in a future release.

Open the DevSpaces Workspace

Open the workspace from the devfile URL:

https://<devspaces-url>/#{github-repo}.git

Replace <devspaces-url> with your DevSpaces instance URL (e.g., devspaces.apps.your-cluster.example.com).

Wait for the workspace to start. Verify Goose is available:

goose --version
goose version 0.x.x
