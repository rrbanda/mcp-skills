The Architecture: DevSpaces + Agent Sandbox

The Enterprise Gap

Cloud sandbox providers solve execution isolation for SaaS, but enterprises need more:

  • On-premises execution — data never leaves the datacenter

  • Existing identity/auth — OIDC, LDAP, RBAC integration

  • Compliance — audit trails, data residency, regulatory requirements

  • Integration — CI/CD, GitOps, container registries

  • Multi-tenancy — shared clusters with tenant isolation

The DevSpaces + Agent Sandbox architecture addresses these requirements on OpenShift.

What Each Component Provides

Component Capabilities

DevSpaces

IDE (Che Code), devfiles, OIDC auth, dashboard, project clone, persistent storage

Coding Agent / Goose

Code understanding, planning, generation, tool use

Agent Sandbox

Isolated execution with VM isolation (Kata/gVisor), NetworkPolicy, lifecycle management

SandboxWarmPool

Pre-warmed pods for sub-second allocation

SandboxTemplate

Allowed resources, network rules, runtime class

Combined Architecture

OpenShift Cluster
├── DevSpaces Workspace Pod
│   ├── Che Code IDE
│   ├── Goose agent (planning, credentials, /projects persistent)
│   └── ────────────────────────────────────────────────────────►
│                                                               │
│   Agent Sandbox Pod ◄─────────────────────────────────────────┘
│   ├── Kata/gVisor runtime
│   ├── Code execution
│   ├── No credentials (automountServiceAccountToken: false)
│   ├── Network locked (deny-all + DNS only)
│   └── Disposable (replaced from WarmPool)
│
└── SandboxWarmPool
    └── Pre-warmed pods ready for instant allocation

How the Agent Uses the Sandbox

Three approaches, from simplest to most integrated:

  1. oc exec — The agent runs oc exec into a sandbox pod. Simplest; works with any agent.

  2. Agent Sandbox Python SDK — Programmatic claim/exec/release lifecycle.

  3. MCP server — An MCP server wraps the SDK; the agent invokes tools that delegate to the sandbox.

Verify Your Environment

Before proceeding to Module 4, verify DevSpaces and (if installed) Agent Sandbox:

oc get pods -n openshift-devspaces | head -5
NAME                                  READY   STATUS    RESTARTS   AGE
devspaces-operator-xxxxx-yyyyy        1/1     Running   0          2d
postgres-xxxxx-yyyyy                  1/1     Running   0          2d
oc get sandboxtemplates -A 2>/dev/null || echo "Agent Sandbox not yet installed"
Agent Sandbox not yet installed
This is expected — Agent Sandbox is installed in Module 4.
